Detailed Description

Object that gets notified from events.

A listener is an object that is notified when an event (or different ones) are called. Every time you want to receive an event, you must use a listener. The best practice is to aggregate a Listener to the object you want to use events with (Note that anything that inherits from the base Object class will have an empty listener).

Using a listener is very simple: you only need to use two functions: the Subscribe and Unsubscribe. The subscribe function will add a function that will be called when the event is dispatched and the unsubscribe function will remove a function or an entire event from the callback list.

The destructor will automatically unsubscribe from all remaining events.

Subscribes a function to an specific event.

Template Parameters
TEventEvent you want to subscribe to
Parameters
nameName of the callback function
callbackFunction you want to use as callback
priorityOrder in which the function will be called

When you want to subscribe to an event, you need to put that event type as the template and then pass a name and function. This will be added to the m_callback map that is stored on the listener. The name is used for unsubscribing only from one function as comparing std::functions (specially lambdas) is very error prone.

When you create an event callback, it MUST have this format

bool OpenPauseMenu(KeyPressedEvent* e);

The most important things to look here are the return value and the arguments.

  • The return value should be a bool that tells if the event has been handled or not. Returning false will make the event system continue to dispatch the event and propagate to the listeners that are after this functions. Returning true will make the system destroy the event and not continue passing it to other members.
  • The parameter is a pointer to the event type we want to subscribe the function. DO NOT include any other parameter

Optionally you can also include a priority to the subscribe, that will ensure this function is called before other the other ones. Callbacks with the highest number will be called before than callbacks with the lowest number. Note that priorities goes from 0 to 255. The default priority is 1 so you can still put priority 0 if you want an event to always be last.

Note
It is important to know that priorities work on a Listener level, not a callback level, so when changing the priority all the callbacks of that specific listener will be set to that priority
